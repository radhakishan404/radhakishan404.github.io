Career-Ops The first job search system that behaves like an operating model, not a spreadsheet.

Career-Ops is an open-source, multi-agent job search system built around AI coding CLIs. It does not mass-apply for you. It does something more useful: it helps you scan, evaluate, personalize, generate ATS-ready PDFs, and track the pipeline so you spend time on the roles that are actually worth your energy.

What it is

Career-Ops is not an auto-apply bot. That is exactly why it is interesting.

A lot of AI job search tools optimize for volume. Career-Ops optimizes for selection quality. That is an important difference. The system reads your CV, profile, proof points, tracked companies, and portal configuration, then uses an AI coding CLI to evaluate a role in context rather than doing naive keyword matching.

From the repo’s own framing, the goal is to help candidates use AI to choose companies, not to blindly spam applications. That creates a better mental model: this is a decision system, a personalization system, and a pipeline system bundled together.

What it is not

A browser bot that blindly applies everywhere, inflates activity metrics, and burns your time on roles that were never a fit.

What it is

A structured evaluation and job-search operations layer that helps you score, personalize, generate, and track before you decide to apply.

The repo strongly recommends against applying to anything below roughly 4.0 / 5. That philosophy is one of the strongest parts of the project.
Who it fits

This is for serious searches, not casual browsing.

Career-Ops makes the most sense if you are actively pursuing a defined transition: senior IC roles, AI roles, platform roles, technical leadership roles, or any search where the cost of applying badly is high. If you have a messy CV, unclear positioning, and no idea what kind of roles you want, the system can still help, but only after you put in some profile work first.

Best
For candidates with a clear target role, real proof points, and enough discipline to review every recommendation manually.
Weak
For people who want “press button, get hired” automation. That is not what this system is designed for.
The system gets stronger when it knows you well. The README is explicit about this: the first evaluations may be mediocre until you feed it your context, story, superpowers, preferences, and anti-goals.
Workflow

What happens when you paste a job URL or description

The architecture is clean enough to explain in one sentence: input goes in, archetype detection happens, a multi-block evaluation runs, then report, PDF, and tracker outputs are generated.

1

Input

You paste a job URL or JD text. The system extracts content directly or works from pasted text.

2

Archetype detection

The job is classified into an internal archetype so scoring and framing are role-aware, not generic.

3

Six-block evaluation

Role summary, CV match, level strategy, compensation research, personalization plan, and interview prep all get generated in one run.

4

Artifacts

You get a markdown report, an ATS-oriented PDF resume, and a tracker entry that fits into the broader pipeline.

The architecture doc also shows a strong batch story: multiple offers can be processed in parallel through headless workers, which matters if your funnel is large and you do not want to manually triage every listing one by one.
Setup

The setup is not hard, but it assumes you can operate a real project

Career-Ops is not a consumer SaaS. You are installing a project, editing config, providing your own CV, and letting an AI coding CLI work inside that directory. That is a feature, not a bug, but it means the right user is someone comfortable with repositories and files.

git clone https://github.com/santifer/career-ops.git
cd career-ops && npm install
npx playwright install chromium
npm run doctor
1

Clone and install

Node.js is required for the scripts and PDF generation. Playwright Chromium is required for PDF output.

2

Copy the profile template

Use config/profile.example.yml as the base and turn it into your actual identity source of truth.

3

Add your CV

Create cv.md in the project root. This is the main context file the system keeps returning to.

4

Customize portals

Copy templates/portals.example.yml to portals.yml and trim it to your actual role keywords, exclusions, and tracked companies.

5

Open your AI CLI in the directory

At that point, the system wants you to work inside Claude Code or Gemini CLI and guide it through your career-specific setup.

One subtle but smart design choice: the project is built to be customized by the agent itself. You ask Claude to change archetypes, update profile data, or edit target companies, and it works on the same files the system runs on.
Customize first

If you skip customization, the output will feel generic

The customization guide makes it clear where the leverage is. If I were onboarding this for myself, I would spend the first hour cleaning these areas before running serious evaluations.

  • config/profile.yml — candidate identity, location, target roles, compensation, narrative.
  • modes/_profile.md — archetype mapping and how your projects get framed against specific role types.
  • portals.yml — job title filters, negative filters, target companies, and board search queries.
  • templates/cv-template.html — if the default PDF look is not your style, this is where it changes.
  • article-digest.md — optional but useful if your strongest proof points live in shipped writing or portfolio work.
If you leave the archetypes untouched while targeting a different market or role family, the scoring logic may be directionally wrong even if the system feels confident.

Gemini CLI support is not a side note here. The project documents native Gemini CLI usage and also offers a standalone API-script path. That makes the repo more durable for people who do not want to be locked into exactly one AI coding interface.
There is also a Go dashboard TUI for browsing the pipeline visually, with filters and status views. That is a strong touch for anyone whose search has already moved beyond a handful of opportunities.
Strengths

What makes Career-Ops better than most “AI for job search” ideas

Three things stand out technically and strategically.

1
It treats the job search as a pipeline with integrity checks, normalization, merging, and canonical states.
2
It keeps a human in the loop and explicitly avoids autonomous application spam.
3
It is configurable enough that the system can become specific to your profile rather than forcing generic templates.
4
It bundles evaluation, resume generation, tracker management, and scanner workflows instead of scattering them across disconnected tools.
The batch architecture is especially interesting. Many repos stop at “single input, single answer.” Career-Ops actually has a serious opinion about orchestration, workers, state, retries, and pipeline output.
Limits

This still depends on your judgment, your writing, and your honesty

There are a few caveats worth being explicit about.

  • The project can personalize, but it cannot invent real proof points you do not have.
  • If your CV is weak or inconsistent, every downstream output inherits that weakness.
  • Compensation research and fit evaluation are useful approximations, not guarantees.
  • Portal scanning breadth is helpful, but you still need to decide where to focus and what to ignore.
  • The more specialized your target role, the more important profile tuning becomes.
Treat every generated PDF and evaluation as a high-quality draft. Do not outsource final accountability to the system just because the output looks polished.
Verdict

My take: this is one of the more mature open-source “AI for work” repos because it has an operating model behind it

What I like most is that Career-Ops is grounded in a real problem and a real outcome. It is not pretending AI magically gets you hired. It is building a system around selection, personalization, and pipeline control.

If you are a serious candidate, especially in technical roles, this repo is worth reading even if you never adopt it fully. The structure alone will improve how you think about job search operations. And if you do adopt it, the real win is not the automation. The real win is that it forces your search to become explicit, reviewable, and strategic.

High-signal repo. Strong framing. Worth studying even if you only steal the operating principles and build your own lighter version.
