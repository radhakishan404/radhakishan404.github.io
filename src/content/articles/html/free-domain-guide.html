01 — Overview

What are these repos?

Both is-a-dev/register and open-domains/register are open-source GitHub repositories maintained by the developer community. They let you register a free custom subdomain by submitting a simple JSON file via a GitHub pull request.

No sign-up. No dashboard. No payment. You just edit a file, raise a PR, and within 24-48 hours your domain is live and pointing to your website.

Yeh repos kya hain?

Dono is-a-dev/register aur open-domains/register open-source GitHub repositories hain jo developer community maintain karti hai. Yeh tumhe ek free custom subdomain register karne dete hain — bas ek simple JSON file GitHub pull request se submit karke.

Koi sign-up nahi. Koi dashboard nahi. Koi payment nahi. Bas ek file edit karo, PR raise karo, aur 24-48 ghante mein tumhara domain live ho jaata hai.

03 — is-a-dev Setup Guide

How to get your .is-a.dev domain

Follow these 6 steps exactly. Takes about 10 minutes to set up. Review takes 24–48 hours.

Apna .is-a.dev domain kaise lo

Yeh 6 steps exactly follow karo. Setup mein lagbhag 10 minutes lagte hain. Review mein 24–48 ghante lagte hain.

1
Fork the repository
Repository fork karo

Go to github.com/is-a-dev/register and click the Fork button in the top-right corner. This creates a copy of the repo in your own GitHub account. Make sure you are logged into your GitHub account first.

github.com/is-a-dev/register pe jao aur top-right corner mein Fork button click karo. Yeh repo ki ek copy tumhare GitHub account mein bana deta hai. Pehle apne GitHub account mein login karo.

2
Go to the domains/ folder
domains/ folder mein jao

In your forked repo, open the domains/ folder. This is where all registered domain files live. Click Add file → Create new file.

Apne forked repo mein domains/ folder open karo. Yahan sab registered domain files rehti hain. Add file → Create new file click karo.

3
Name your file correctly
File ka naam sahi rakho

The file name becomes your subdomain. If you want yourname.is-a.dev, name the file yourname.json. Use only lowercase letters, numbers, and hyphens. No spaces.

File ka naam hi tumhara subdomain banega. Agar tumhe yourname.is-a.dev chahiye, toh file ka naam yourname.json rakho. Sirf lowercase letters, numbers, aur hyphens use karo. Spaces nahi.

Check first if your desired subdomain is already taken by searching in the domains/ folder. Pehle check karo ki tumhara chahuta subdomain already liya gaya hai ya nahi — domains/ folder mein search karo.
4
Add your JSON content
JSON content add karo

Paste the following JSON into your new file. Replace all placeholder values with your own details.

Yeh JSON niche di gayi file mein paste karo. Sab placeholder values apni details se replace karo.

yourname.json 
{
  "description": "Your name — Developer Portfolio",
  "repo": "https://github.com/yourusername/yourrepo",
  "owner": {
    "username": "yourgithubusername",
    "email": "your@email.com"
  },
  "record": {
    "CNAME": "yourgithubusername.github.io"
  }
}
CNAME is for GitHub Pages sites. If your site is on Vercel or Netlify, the CNAME value will be different — check their custom domain docs for the exact value. CNAME GitHub Pages sites ke liye hai. Agar tumhari site Vercel ya Netlify pe hai, toh CNAME value alag hogi — unke custom domain docs check karo exact value ke liye.

Using Vercel instead of GitHub Pages?

GitHub Pages ki jagah Vercel use kar rahe ho?

for vercel 
{
  "description": "Your name — Portfolio",
  "repo": "https://github.com/yourusername/yourrepo",
  "owner": {
    "username": "yourgithubusername",
    "email": "your@email.com"
  },
  "record": {
    "CNAME": "cname.vercel-dns.com"
  }
}
5
Commit and open a Pull Request
Commit karo aur Pull Request open karo

Scroll down and click Commit new file. Then GitHub will show a prompt — click Open Pull Request. Add a clear title like: "Register yourname.is-a.dev for my portfolio". Then click Create pull request.

Scroll down karo aur Commit new file click karo. Phir GitHub ek prompt dikhayega — Open Pull Request click karo. Ek clear title add karo jaise: "Register yourname.is-a.dev for my portfolio". Phir Create pull request click karo.

Important: The is-a-dev team warns against AI-generated PRs. Write your PR description yourself in simple English. Say what your site is about and why you want this domain. Important: is-a-dev team AI-generated PRs accept nahi karte. Apna PR description khud likho simple English mein. Batao tumhari site kiske baare mein hai aur yeh domain kyun chahiye.
6
Wait for review and go live
Review ka wait karo aur live ho jao

Maintainers review your PR within 24–48 hours. If changes are requested, make them quickly. Once your PR is merged, your DNS records go live within a few minutes. Your domain is ready.

Maintainers tumhara PR 24–48 ghante mein review karte hain. Agar changes request kiye gaye, unhe jaldi karo. Ek baar PR merge ho jaata hai, tumhare DNS records kuch minutes mein live ho jaate hain. Tumhara domain ready hai.

After merge, go to your GitHub Pages repo → Settings → Pages → Custom domain, add your new domain, and enable Enforce HTTPS. Merge ke baad, apne GitHub Pages repo mein jao → Settings → Pages → Custom domain, apna naya domain add karo aur Enforce HTTPS enable karo.
04 — open-domains Setup Guide

How to get an open-domains subdomain

The process is almost identical to is-a-dev. The main difference is the repo URL and the available domain extensions.

open-domains subdomain kaise lo

Process almost is-a-dev jaisi hi hai. Main difference repo URL aur available domain extensions hain.

1
Fork open-domains/register
open-domains/register fork karo

Go to github.com/open-domains/register and click Fork. Same as before.

github.com/open-domains/register pe jao aur Fork click karo. Pehle jaisa hi.

2
Choose your domain type and create the file
Domain type choose karo aur file banao

In the forked repo, navigate to domains/is-cool.dev/ (or whichever domain type you want). Create a new file named yourname.json inside that folder.

Forked repo mein domains/is-cool.dev/ folder mein jao (ya jo bhi domain type tumhe chahiye). Uss folder ke andar yourname.json naam ki naye file banao.

3
Add the JSON — same format
JSON add karo — same format

The JSON structure is the same as is-a-dev. Copy the template from step 4 above and update your details.

JSON structure is-a-dev jaisi hi hai. Upar step 4 ka template copy karo aur apni details update karo.

yourname.json 
{
  "description": "Your name — Developer Portfolio",
  "repo": "https://github.com/yourusername/yourrepo",
  "owner": {
    "username": "yourgithubusername",
    "email": "your@email.com"
  },
  "records": {
    "CNAME": "yourgithubusername.github.io"
  }
}
Note: open-domains uses "records" (plural) while is-a-dev uses "record" (singular). Don't mix them up. Note: open-domains mein "records" (plural) hai jabki is-a-dev mein "record" (singular) hai. Inhe mix mat karo.
4
Commit and raise a Pull Request
Commit karo aur Pull Request raise karo

Same as is-a-dev — commit the file, then open a pull request with a clear description of your site. Wait for the maintainers to review and merge.

is-a-dev jaisa hi — file commit karo, phir apni site ki clear description ke saath pull request open karo. Maintainers ke review aur merge ka wait karo.

05 — Connecting to GitHub Pages

Connect your new domain to your site

Once your PR is merged, do these steps to make your domain actually point to your website.

Apna naya domain site se connect karo

Ek baar PR merge ho jaaye, yeh steps karo taaki tumhara domain actually tumhari website pe point kare.

1
Go to your GitHub Pages repository settings
Apne GitHub Pages repository settings mein jao

Open your portfolio repo on GitHub. Click Settings in the top menu bar. Scroll down to the Pages section in the left sidebar.

Apna portfolio repo GitHub pe open karo. Top menu bar mein Settings click karo. Left sidebar mein Pages section tak scroll karo.

2
Add your custom domain
Custom domain add karo

In the Custom domain field, type your new domain (e.g. yourname.is-a.dev) and click Save. GitHub will verify it automatically — this takes a minute or two.

Custom domain field mein apna naya domain type karo (jaise yourname.is-a.dev) aur Save click karo. GitHub automatically verify karega — isme ek-do minute lagte hain.

3
Enable HTTPS
HTTPS enable karo

Once verification is complete, tick the Enforce HTTPS checkbox below the custom domain input. Your site now loads on HTTPS — secure and professional.

Verification complete hone ke baad, custom domain input ke neeche Enforce HTTPS checkbox tick karo. Tumhari site ab HTTPS pe load hogi — secure aur professional.

Your site is now live at yourname.is-a.dev. DNS can take up to a few hours to propagate globally — if you see a 404, wait and try again. Tumhari site ab yourname.is-a.dev pe live hai. DNS globally propagate hone mein kuch ghante lag sakte hain — agar 404 dikhaye, wait karo aur dobara try karo.
06 — Tips & Common Errors

Things to keep in mind

Kuch important baatein

Common mistakes to avoid

Common mistakes jo avoid karne chahiye

Don't use AI to generate your PR. The is-a-dev team explicitly warns about this and will reject or block AI-generated pull requests. Write it yourself. PR generate karne ke liye AI mat use karo. is-a-dev team explicitly iske baare mein warn karti hai aur AI-generated pull requests reject ya block kar deti hai. Khud likho.
Your site must have content. Empty pages or "coming soon" pages will be rejected. Your portfolio should have something real — even a basic page works. Tumhari site mein content hona chahiye. Empty pages ya "coming soon" pages reject ho jaate hain. Tumhara portfolio mein kuch real hona chahiye — even basic page bhi kaam karta hai.
Don't use Netlify with is-a-dev. open-domains also has issues with Netlify subdomain verification. Use GitHub Pages or Vercel instead. is-a-dev ke saath Netlify mat use karo. open-domains ko bhi Netlify subdomain verification mein issues hain. GitHub Pages ya Vercel use karo.

Quick answers

Quick answers

How long does approval take?
Approval mein kitna time lagta hai?
Usually 24–48 hours. Keep an eye on your PR for review comments and respond quickly if changes are requested.
Usually 24–48 ghante. Apna PR review comments ke liye dekhte raho aur agar changes request hue toh jaldi respond karo.
Is it really free forever?
Kya yeh sach mein hamesha ke liye free hai?
Yes. Both repos are community-maintained and free. is-a-dev is sponsored by Cloudflare's Project Alexandria, so it's stable and well-supported.
Haan. Dono repos community-maintained aur free hain. is-a-dev Cloudflare ke Project Alexandria se sponsored hai, toh stable aur well-supported hai.
Can I update or delete my domain later?
Kya main baad mein domain update ya delete kar sakta hoon?
Yes. To update — edit your JSON file in the domains/ folder via another PR. To delete — delete the file via a PR with a note explaining why.
Haan. Update karne ke liye — domains/ folder mein apni JSON file another PR se edit karo. Delete karne ke liye — ek PR se file delete karo aur reason explain karo.
I see a 404 after my PR was merged. What do I do?
PR merge hone ke baad 404 aa raha hai. Kya karoon?
Go to your portfolio repo → Settings → Pages → Custom Domain. Add your domain there and enable Enforce HTTPS. DNS propagation can also take a few hours — wait and try again.
Apne portfolio repo mein jao → Settings → Pages → Custom Domain. Wahan apna domain add karo aur Enforce HTTPS enable karo. DNS propagation mein kuch ghante lag sakte hain — wait karo aur dobara try karo.
